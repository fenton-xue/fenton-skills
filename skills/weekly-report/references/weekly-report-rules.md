# 测试周报规则

## 日历模型

- 用户是软件测试工程师。日历活动表示需求排期。
- 一个需求有三个日历活动，三个活动标题完全一致：
  - 编写用例
  - 初测
  - 复测
- 阶段必须来自活动颜色：
  - 石墨黑 / Google 活动颜色 `8` / `#e1e1e1` = 编写用例
  - 孔雀蓝 / Google 活动颜色 `9` / `#5484ed` = 初测
  - 鼠尾草绿 / Google 活动颜色 `10` / `#51b749` = 复测
- 不要根据活动标题推断阶段。
- Google Calendar 连接器只用于获取排期；事件颜色必须通过 Chrome 插件接管 Google Calendar 页面，并配合 `scripts/extract_calendar_event_colors.mjs` 处理。
- 取色脚本会记录页面中的中文颜色名、Google 颜色 ID、十六进制颜色或 `rgb(...)`，并输出可传给周报生成脚本的事件数据。
- 跨月或跨周活动仍然是一条活动，不要按月份拆分或让模型拼接；开始/结束日期以 Google Calendar 连接器返回值为准。
- 取色脚本需要把 Chrome 页面上看到的跨月活动片段按事件 ID 或日期重叠匹配回连接器事件。若同一工单有多个阶段且页面缺少事件 ID/日期范围，不能把无日期颜色强行匹配到某个阶段，应要求人工补充。
- 过往活动在页面上会被降低亮度。取色脚本必须按原始色相/颜色类别归一化：
  - 偏灰/黑 = 石墨黑 = 编写用例
  - 偏蓝 = 孔雀蓝 = 初测
  - 偏绿 = 鼠尾草绿 = 复测

## 周区间

- 工作周按周一到周五计算。
- 生成以下三个时间段：
  - 本周：`today` 所在周
  - 下周：`today + 1 周`
  - 下下周：`today + 2 周`
- Google Calendar 全天活动的 `end` 是排除式结束日期。例如 `start=2026-05-29T00:00:00`、`end=2026-05-30T00:00:00` 表示活动结束于 `2026-05-29`。

## 本周工作内容 / 下周工作计划

目标工作周内有重叠工作日的需求活动，都要在 CSV 中生成一行：

```txt
需求名称,负责人,进度,（预计）完成时间
```

进度规则：

- 如果需求的上线日期在今天及以前，进度单元格填写 `已上线`。该规则优先级最高，覆盖 `复测:100%` 等阶段进度。
- 如果活动在目标周内或目标周之前结束，输出 `{事项}:100%`。
- 如果活动在目标周内结束不了，按工作日计算进度：
  - 分子 = 活动开始日期到目标周周五之间的工作日数
  - 分母 = 活动开始日期到活动结束日期之间的全部工作日数
  - 进度 = 分子 / 分母 * 100
  - 按用户示例执行：`4/(4+2)=66.7%` 输出 `70%`，即四舍五入到最接近的 10%。
- 进度列的事项名称显示规则：
  - 编写用例仍显示为 `编写用例`
  - 初测显示为 `UAT`
  - 复测显示为 `PRE`
- 如果同一需求在同一目标周内有多个阶段，保留一行，并按阶段顺序用 `、` 拼接，例如 `编写用例:100%、UAT:40%`。

完成时间规则：

- 查找该需求在规划范围内，即截至下下周周五结束的复测活动。
- 上线日期 = 复测结束日期所在周的周四。
- 如果上线日期在今天及以前，完成时间填写 `上线时间：YYYY.MM.DD`，同时进度单元格必须填写 `已上线`。
- 如果上线日期在今天之后，完成时间填写 `预计上线时间: YYYY.MM.DD`。
- 以下情况完成时间单元格留空：
  - 截至下下周周五仍未找到复测结束时间
  - 复测结束日期所在周存在占用工作日的节假日
- 留空原因不写入输出文件，只在回复中提醒用户自行补充。

## 双周计划

双周计划只看下周和下下周。

- 只包含初测和复测。
- 不包含编写用例。
- 不体现阶段是否结束，也不体现“未结束”。
- 如果未来两周同一个需求既有初测也有复测，写成 `(初测+复测)`。
- 默认内容：

```txt
薛丰推: 主要测试#89608 海运系统消息中心搭建(初测+复测), #131002 【联测】CMS和DI迁移OHUB(初测)
```

## 文件放置

- 不要在项目根目录创建临时文件。
- 日历原始输入数据不需要留存；优先在内存中处理，或通过标准输入传给脚本。
- 如因工具限制必须写临时输入文件，应放到 `.agent/weekly-report/`，生成周报后立即删除。
- 最终周报草稿保存为 `.agent/weekly-report/weekly-report-YYYYMMDD.csv`，例如 `.agent/weekly-report/weekly-report-20260529.csv`。
- 所有文件使用 UTF-8 编码。

## CSV 输出结构

- CSV 文件包含多个区块：`一. 本周工作内容`、`二. 下周工作计划`、`三. 双周计划`。
- 本周和下周区块表头固定为：`需求名称,负责人,进度,（预计）完成时间`。
- 双周计划区块表头固定为：`内容`。
- 不在 CSV 中输出额外说明区；需要用户补充的空值只在回复中提醒。
